EidolonLegacy CraftTweaker 2 使用指南
适用环境：Minecraft 1.12.2 / CraftTweaker 2 / EidolonLegacy
本文说明 EidolonLegacy 当前开放给 CraftTweaker 2 的脚本接口。
每个系统先给出导入、方法、参数说明，再给出可以直接复制修改的示例。
脚本通常放在当前实例的 scripts 目录，例如：.minecraft/scripts/eidolon_legacy.zs
建议所有自定义 id 都使用自己的命名空间，例如：
    pack:test_recipe
    ctrecipes:cosmic_meatballs_crucible
一、通用规则
============

1. 导入接口

   每个脚本文件里需要先导入要使用的接口。例如：

       import mods.eidolon.Worktable;
       import mods.eidolon.Crucible;
       import mods.eidolon.Altar;
       import mods.eidolon.Athame;
       import mods.eidolon.Research;
       import mods.eidolon.Incubator;

   导入后可以直接写 Worktable.addRecipe(...)。
   如果不导入，也可以写完整类名，例如 mods.eidolon.Worktable.addRecipe(...)。

2. id 写法

   id 是配方、仪式、研究或规则的唯一标识，建议写成：

       "命名空间:名称"

   示例：

       "pack:test_worktable"
       "mymod:custom_ritual"

   替换内置内容时，先用 removeById 删除原 id，再添加新内容。

3. 物品和材料写法

   常用 CraftTweaker 写法：

       <minecraft:diamond>        表示 1 个钻石
       <minecraft:dye:4>          表示 metadata 为 4 的染料
       <minecraft:paper> * 2      表示 2 张纸
       <liquid:water> * 1000      表示 1000 mB 水，也就是一桶水
       null                       表示空槽，或者可选参数不传

   IItemStack 一般用于“实际输出物品”。
   IIngredient 一般用于“可匹配的输入材料”，可以是物品、矿辞、带 metadata 的物品等。

4. 删除规则

   优先使用 removeById 删除指定内容。
   removeByOutput 会删除所有输出匹配的内容，范围更大。
   removeAll 会清空整个系统，正式整合包中只建议在完全重写该系统时使用。

5. 调试入口

   脚本加载失败时先看：

       run/client/crafttweaker.log

   如果 HEI 或 Codex 不显示新增内容，先确认 crafttweaker.log 没有语法错误，再看 latest.log 是否有注册或运行时报错。

   烟雾测试脚本正常加载时，crafttweaker.log 中应出现类似内容：

       Loading Eidolon CraftTweaker2 smoke test script
       Loaded Eidolon CraftTweaker2 smoke test script


二、Worktable：魔法工作台配方
============================

接口类：

    mods.eidolon.Worktable

导入：

    // 导入 EidolonLegacy 的 Worktable CT 接口。
    // 没有这一行，就不能直接调用 Worktable.addRecipe、Worktable.removeById 等方法。
    import mods.eidolon.Worktable;

用途：

    添加、删除魔法工作台配方。
    Worktable 配方包含一个 3x3 主输入区和 4 个试剂槽。

方法：

    Worktable.addRecipe(String id, IItemStack output, IIngredient[] grid, IIngredient[] reagents);
    Worktable.removeById(String id);
    Worktable.removeByOutput(IIngredient output);
    Worktable.removeAll();

参数：

    id        配方 id。
    output    输出物品。
    grid      3x3 主输入，必须正好 9 个元素。顺序是从左到右、从上到下。
    reagents  4 个试剂槽，必须正好 4 个元素。

添加配方示例：

    // 删除同 id 的旧配方，避免重复添加或覆盖顺序不清楚。
    Worktable.removeById("pack:test_worktable");

    // 添加一条 Worktable 配方。
    Worktable.addRecipe(
        // 第 1 个参数：配方 id。
        "pack:test_worktable",

        // 第 2 个参数：输出物品。
        // 这里表示合成结果是 1 个测试符印。
        <eidolon:test_sigil>,

        // 第 3 个参数：3x3 主输入。
        // 必须正好写 9 个元素，顺序是从左到右、从上到下。
        [
            // 第 1 行第 1 格：左上角。
            <minecraft:stick>,
            // 第 1 行第 2 格：上方中间为空。
            null,
            // 第 1 行第 3 格：右上角为空。
            null,

            // 第 2 行第 1 格：左侧中间为空。
            null,
            // 第 2 行第 2 格：正中心。
            <minecraft:diamond>,
            // 第 2 行第 3 格：右侧中间为空。
            null,

            // 第 3 行第 1 格：左下角为空。
            null,
            // 第 3 行第 2 格：下方中间为空。
            null,
            // 第 3 行第 3 格：右下角。
            <minecraft:stick>
        ],

        // 第 4 个参数：4 个试剂槽。
        // 必须正好写 4 个元素，null 表示该试剂槽为空。
        [
            // 试剂槽 1：灵魂碎片。
            <eidolon:soul_shard>,
            // 试剂槽 2：空。
            null,
            // 试剂槽 3：空。
            null,
            // 试剂槽 4：白镴嵌片。
            <eidolon:pewter_inlay>
        ]
    );

删除配方示例：

    // 按配方 id 删除。
    Worktable.removeById("eidolon:athame");

    // 按输出物品删除。所有输出匹配该物品的 Worktable 配方都会被删除。
    Worktable.removeByOutput(<eidolon:reaper_scythe>);

    // 删除全部 Worktable 配方。谨慎使用。
    // Worktable.removeAll();


三、Crucible：坩埚配方
=====================

接口类：

    mods.eidolon.Crucible

导入：

    // 导入 EidolonLegacy 的 Crucible CT 接口。
    import mods.eidolon.Crucible;

用途：

    添加、删除坩埚分步骤配方。
    坩埚配方由多个步骤组成，每一步可以投入多个物品，并可要求搅拌指定次数。

方法：

    Crucible.addRecipe(String id, IItemStack output, int[] stirs, IIngredient[][] steps, @Optional IIngredient stirrer, @Optional ILiquidStack fluid);
    Crucible.removeById(String id);
    Crucible.removeByOutput(IIngredient output);
    Crucible.removeAll();

参数：

    id       配方 id。
    output   最终输出物品。
    stirs    每一步需要搅拌的次数。
    steps    每一步投入的物品。外层数组是步骤，内层数组是该步骤的投入物。
    stirrer  搅拌物，可省略；默认是木棍。
    fluid    起始流体，可省略；默认是 1000 mB 水。

限制：

    stirs 的数量必须和 steps 的步骤数量一致。
    每一步最多 6 个投入物。
    同一步内的投入物不区分顺序。
    步骤之间必须按顺序完成。
    HEI 和 Codex 每页显示 5 步，长配方可以翻页查看。

添加配方示例：

    // 删除同 id 的旧配方。
    Crucible.removeById("pack:test_crucible");

    // 添加一条 Crucible 配方。
    Crucible.addRecipe(
        // 第 1 个参数：配方 id。
        "pack:test_crucible",

        // 第 2 个参数：坩埚最终产物。
        <eidolon:death_essence>,

        // 第 3 个参数：每一步需要搅拌的次数。
        // 数组长度必须等于下面 steps 的步骤数。
        [
            // 第 1 步：不需要搅拌。
            0,
            // 第 2 步：需要搅拌 2 次。
            2,
            // 第 3 步：需要搅拌 1 次。
            1
        ],

        // 第 4 个参数：每一步投入的物品。
        // 外层数组表示步骤，内层数组表示该步骤要投入的物品。
        [
            // 第 1 步：投入腐肉和灵魂碎片。
            [
                <minecraft:rotten_flesh>,
                <eidolon:soul_shard>
            ],

            // 第 2 步：投入骨头和红石粉。
            [
                <minecraft:bone>,
                <minecraft:redstone>
            ],

            // 第 3 步：投入木炭。
            [
                <minecraft:coal:1>
            ]
        ],

        // 第 5 个参数：搅拌物。
        // 需要搅拌的步骤必须手持该物品右键坩埚。
        <minecraft:stick>,

        // 第 6 个参数：起始流体。
        // 这里表示一桶水。
        <liquid:water> * 1000
    );

省略可选参数示例：

    // 不写 stirrer 和 fluid 时，使用默认搅拌物和默认起始流体。
    Crucible.addRecipe(
        "pack:test_crucible_defaults",
        <minecraft:slime_ball>,
        [0],
        [
            [
                <minecraft:wheat_seeds>,
                <eidolon:soul_shard>
            ]
        ]
    );

删除配方示例：

    // 按配方 id 删除。
    Crucible.removeById("eidolon:ender_calx");

    // 按输出物品删除。
    Crucible.removeByOutput(<eidolon:shadow_gem>);

    // 删除全部坩埚配方。谨慎使用。
    // Crucible.removeAll();


四、Altar Ritual：祭坛仪式
=========================

接口类：

    mods.eidolon.Altar

导入：

    // 导入 EidolonLegacy 的 Altar CT 接口。
    // 祭坛仪式和祭坛供物都使用这个类。
    import mods.eidolon.Altar;

用途：

    添加、删除祭坛仪式。

通用概念：

    capacity   祭坛容量需求。
    power      祭坛力量需求。
    sacrifice  火盆祭品。
    offerings  Stone Hand 上依次消耗的物品。
    focus      Necrotic Focus 中的焦点物品。
    health     玩家生命代价，20.0 等于 10 颗心。

仪式方法：

    Altar.addItemResult(id, output, capacity, power, offerings, sacrifice);
    Altar.addTransform(id, output, focus, capacity, power, offerings, health, sacrifice);
    Altar.addCharge(id, output, focus, capacity, power, offerings, sacrifice);
    Altar.addSummon(id, displayOutput, entityId, focus, capacity, power, offerings, sacrifice);
    Altar.addAbsorption(id, output, focus, capacity, power, offerings, sacrifice);
    Altar.addSanguine(id, output, focus, sacrifice, capacity, power, health, offerings);
    Altar.addPurify(id, sacrifice, capacity, power, offerings, focus);
    Altar.addCrystal(id, sacrifice, capacity, power, offerings, focus);
    Altar.addAllure(id, sacrifice, capacity, power, offerings, focus);
    Altar.addRepelling(id, sacrifice, capacity, power, offerings, focus);
    Altar.addDeceit(id, sacrifice, capacity, power, offerings, focus);
    Altar.addDaylight(id, sacrifice, capacity, power, offerings, focus);
    Altar.addMoonlight(id, sacrifice, capacity, power, offerings, focus);
    Altar.removeById(id);
    Altar.removeByOutput(output);
    Altar.removeAll();

通用材料数组示例：

    // 常用 Stone Hand 消耗物。
    val altarOfferings as crafttweaker.item.IIngredient[] = [
        <eidolon:soul_shard>,
        <minecraft:bone>
    ];

    // 场域仪式常用消耗物。
    val fieldOfferings as crafttweaker.item.IIngredient[] = [
        <eidolon:soul_shard>,
        <eidolon:soul_shard>
    ];

1. 普通产物仪式

    方法：

        Altar.addItemResult(id, output, capacity, power, offerings, sacrifice);

    示例：

        // 添加一条普通产物仪式。
        Altar.addItemResult(
            // 第 1 个参数：仪式 id。
            "pack:test_altar_result",

            // 第 2 个参数：仪式完成后生成的物品。
            <eidolon:death_essence>,

            // 第 3 个参数：容量需求。
            3.0,

            // 第 4 个参数：力量需求。
            3.0,

            // 第 5 个参数：Stone Hand 上依次消耗的物品。
            altarOfferings,

            // 第 6 个参数：火盆祭品。
            <minecraft:rotten_flesh>
        );

2. 转化仪式

    方法：

        Altar.addTransform(id, output, focus, capacity, power, offerings, health, sacrifice);

    示例：

        // 添加一条转化仪式。
        Altar.addTransform(
            // 仪式 id。
            "pack:test_altar_transform",
            // 转化后的物品。
            <eidolon:sapping_sword>,
            // Necrotic Focus 中的被转化物。
            <minecraft:iron_sword>,
            // 容量需求。
            4.0,
            // 力量需求。
            4.0,
            // Stone Hand 消耗物。
            [
                <eidolon:shadow_gem>,
                <eidolon:soul_shard>
            ],
            // 玩家生命消耗。20.0 表示 10 颗心。
            20.0,
            // 火盆祭品。
            <minecraft:redstone>
        );

3. 充能仪式

    方法：

        Altar.addCharge(id, output, focus, capacity, power, offerings, sacrifice);

    示例：

        // 添加一条充能仪式。
        Altar.addCharge(
            // 仪式 id。
            "pack:test_altar_charge",
            // HEI/Codex 中显示的充能后物品。
            <eidolon:soulfire_wand>,
            // Necrotic Focus 中需要充能的物品。
            <eidolon:soulfire_wand>,
            // 容量需求。
            3.0,
            // 力量需求。
            3.0,
            // Stone Hand 消耗物。
            [
                <eidolon:lesser_soul_gem>,
                <minecraft:blaze_powder>
            ],
            // 火盆祭品。
            <minecraft:glowstone_dust>
        );

4. 召唤仪式

    方法：

        Altar.addSummon(id, displayOutput, entityId, focus, capacity, power, offerings, sacrifice);

    示例：

        // 添加一条召唤仪式。
        Altar.addSummon(
            // 仪式 id。
            "pack:test_altar_summon",
            // 显示物品，只用于 HEI/Codex 展示。
            <minecraft:spawn_egg>.withTag({EntityTag: {id: "minecraft:zombie"}}),
            // 实际召唤的实体 id。
            "minecraft:zombie",
            // Necrotic Focus 中的焦点物品。
            <minecraft:rotten_flesh>,
            // 容量需求。
            3.0,
            // 力量需求。
            3.0,
            // Stone Hand 消耗物。
            altarOfferings,
            // 火盆祭品。
            <minecraft:coal:1>
        );

5. 吸收仪式

    方法：

        Altar.addAbsorption(id, output, focus, capacity, power, offerings, sacrifice);

    示例：

        // 添加一条吸收仪式。
        Altar.addAbsorption(
            // 仪式 id。
            "pack:test_altar_absorption",
            // HEI/Codex 中显示的输出。
            <eidolon:summoning_staff>,
            // Necrotic Focus 中的吸收载体。
            <eidolon:summoning_staff>,
            // 容量需求。
            4.0,
            // 力量需求。
            4.0,
            // Stone Hand 消耗物。
            altarOfferings,
            // 火盆祭品。
            <minecraft:bone>
        );

6. 血契制作

    方法：

        Altar.addSanguine(id, output, focus, sacrifice, capacity, power, health, offerings);

    注意：

        addSanguine 的参数顺序和 addTransform 不同。
        它先写 focus 和 sacrifice，再写 capacity、power、health、offerings。

    示例：

        // 添加一条血契仪式。
        Altar.addSanguine(
            // 仪式 id。
            "pack:test_sanguine",
            // 仪式结果。
            <eidolon:sapping_sword>,
            // Necrotic Focus 中的基础物品。
            <minecraft:iron_sword>,
            // 火盆祭品。
            <minecraft:redstone>,
            // 容量需求。
            4.0,
            // 力量需求。
            4.0,
            // 生命代价。
            20.0,
            // Stone Hand 消耗物。
            [
                <eidolon:shadow_gem>,
                <eidolon:soul_shard>,
                <minecraft:ghast_tear>
            ]
        );

7. 场域仪式

    方法：

        Altar.addPurify(id, sacrifice, capacity, power, offerings, focus);
        Altar.addCrystal(id, sacrifice, capacity, power, offerings, focus);
        Altar.addAllure(id, sacrifice, capacity, power, offerings, focus);
        Altar.addRepelling(id, sacrifice, capacity, power, offerings, focus);
        Altar.addDeceit(id, sacrifice, capacity, power, offerings, focus);
        Altar.addDaylight(id, sacrifice, capacity, power, offerings, focus);
        Altar.addMoonlight(id, sacrifice, capacity, power, offerings, focus);

    说明：

        这些仪式通常没有物品产物，主要产生世界效果。
        focus 是可选参数，可以省略。

    示例：

        // 净化仪式。
        Altar.addPurify(
            "pack:test_purify",
            <minecraft:speckled_melon>,
            3.0,
            3.0,
            fieldOfferings
        );

        // 结晶仪式。
        Altar.addCrystal(
            "pack:test_crystal",
            <minecraft:dye:15>,
            2.0,
            2.0,
            fieldOfferings
        );

        // 诱引、驱离、欺瞒、日光、月光仪式。
        Altar.addAllure("pack:test_allure", <minecraft:double_plant:4>, 3.0, 3.0, fieldOfferings);
        Altar.addRepelling("pack:test_repelling", <minecraft:prismarine_crystals>, 3.0, 3.0, fieldOfferings);
        Altar.addDeceit("pack:test_deceit", <minecraft:emerald>, 3.0, 3.0, fieldOfferings);
        Altar.addDaylight("pack:test_daylight", <minecraft:double_plant:0>, 3.0, 3.0, fieldOfferings);
        Altar.addMoonlight("pack:test_moonlight", <minecraft:dye:0>, 3.0, 3.0, fieldOfferings);

    效果速查：

        addAllure     吸引附近被动生物。
        addRepelling  驱离附近敌对生物。
        addDeceit     影响附近村民和村庄声望。
        addDaylight   推进到白天。
        addMoonlight  推进到夜晚。

删除仪式示例：

    // 按仪式 id 删除。
    Altar.removeById("eidolon:lesser_soul_gem");

    // 按显示输出或实际输出删除。
    Altar.removeByOutput(<eidolon:death_essence>);

    // 删除全部祭坛仪式。谨慎使用。
    // Altar.removeAll();


五、Altar Offering：祭坛供物
==========================

接口类：

    mods.eidolon.Altar

用途：

    添加、删除祭坛供物规则。
    供物用于给祭坛提供容量和力量。

供物方法：

    Altar.addOffering(id, item, capacity, power);
    Altar.addBlockOffering(id, item, capacity, power);
    Altar.addPlateOffering(id, item, capacity, power);
    Altar.addOfferingModes(id, item, capacity, power, blockOffering, plateOffering);
    Altar.removeOfferingById(id);
    Altar.removeOfferingByItem(item);
    Altar.removeAllOfferings();

说明：

    addOffering 会根据物品是否是方块物品自动决定模式。
    addBlockOffering 只作为放置在世界里的方块供物生效。
    addPlateOffering 只作为祭品盘里的物品供物生效。
    addOfferingModes 可以手动指定两种模式。
    祭品盘供物只提供容量和力量，不参与仪式合成的 Stone Hand 消耗列表。
    相同 id，也就是相同 key 的供物不会重复叠加。
    不同 key 的供物可以叠加。

添加默认供物示例：

    // 添加一条默认供物规则。
    Altar.addOffering(
        // 供物 id，同时也是数值分组 key。
        "pack:test_offering",
        // 供物物品。
        <minecraft:diamond>,
        // 提供的容量。
        2.0,
        // 提供的力量。
        1.0
    );

仅方块供物示例：

    // 只在物品作为方块放置时生效。
    Altar.addBlockOffering(
        "pack:test_block_offering",
        <minecraft:redstone_torch>,
        0.0,
        2.0
    );

仅祭品盘供物示例：

    // 只在物品放进祭品盘时生效。
    Altar.addPlateOffering(
        "pack:test_plate_offering",
        <minecraft:emerald>,
        1.0,
        2.0
    );

手动指定供物模式示例：

    // 手动指定供物在哪些模式下生效。
    Altar.addOfferingModes(
        // 供物规则 id。
        "pack:test_skull_offering",
        // 供物物品。
        <minecraft:skull:0>,
        // 提供容量。
        2.0,
        // 提供力量。
        0.0,
        // 是否作为方块供物生效。
        true,
        // 是否作为祭品盘供物生效。
        true
    );

删除供物示例：

    // 按供物规则 id 删除。
    Altar.removeOfferingById("eidolon:light");

    // 按物品删除所有相关供物规则。
    Altar.removeOfferingByItem(<minecraft:diamond>);

    // 删除全部供物规则。谨慎使用。
    // Altar.removeAllOfferings();


六、Athame：仪式匕首采集
=======================

接口类：

    mods.eidolon.Athame

导入：

    // 导入 EidolonLegacy 的 Athame CT 接口。
    import mods.eidolon.Athame;

用途：

    添加、删除仪式匕首采集规则。

方法：

    Athame.addRecipe(String id, IIngredient source, IItemStack output, @Optional String sourceLabel);
    Athame.removeById(String id);
    Athame.removeByOutput(IIngredient output);
    Athame.removeBySource(IIngredient source);
    Athame.removeAll();

参数：

    id           采集规则 id。
    source       可被仪式匕首采集的来源。
    output       成功采集后的额外产物。
    sourceLabel  Codex 中显示的来源名称，可省略。

添加采集规则示例：

    // 删除同 id 的旧规则。
    Athame.removeById("pack:test_athame_harvest");

    // 添加一条仪式匕首采集规则。
    Athame.addRecipe(
        // 第 1 个参数：采集规则 id。
        "pack:test_athame_harvest",

        // 第 2 个参数：采集来源。
        // 这里表示红花 metadata 0。
        <minecraft:red_flower:0>,

        // 第 3 个参数：采集产物。
        // 成功时额外掉落灵魂碎片。
        <eidolon:soul_shard>,

        // 第 4 个参数：来源显示名。
        // 会显示在 Codex 的来源说明中。
        "玫瑰"
    );

删除采集规则示例：

    // 按采集规则 id 删除。
    Athame.removeById("eidolon:avennian_sprig_from_fern");

    // 按产物删除。
    Athame.removeByOutput(<eidolon:oanna_bloom>);

    // 按来源删除。
    Athame.removeBySource(<minecraft:waterlily>);

    // 删除全部仪式匕首采集规则。谨慎使用。
    // Athame.removeAll();


七、Research：研究和研究笔记触发
==============================

接口类：

    mods.eidolon.Research

导入：

    // 导入 EidolonLegacy 的 Research CT 接口。
    import mods.eidolon.Research;

用途：

    添加、删除研究。
    添加、删除研究笔记触发来源。

方法：

    Research.addResearch(id, name, stars, prerequisites, stepItems, source, description);
    Research.addResearchLang(id, stars, prerequisites, stepItems, sourceKey);
    Research.removeById(id);
    Research.removeAll();
    Research.addBlockTrigger(blockId, researchId);
    Research.addEntityTrigger(entityId, researchId);
    Research.addDimensionTrigger(dimension, researchId);
    Research.addFluidTrigger(blockId, researchId);
    Research.removeBlockTriggers(blockId);
    Research.removeEntityTriggers(entityId);
    Research.removeDimensionTriggers(dimension);
    Research.removeFluidTriggers(blockId);
    Research.removeAllTriggers();

参数：

    id             研究 id。
    name           研究显示名称，可以直接写中文。
    stars          研究星级，必须为 1 到 5。
    prerequisites  前置研究 id 数组。
    stepItems      每个阶段可提交的物品数组。外层数量必须等于 stars。
    source         Codex 来源文字，可以直接写中文，可省略。
    description    Codex 描述文字，可以直接写中文，可省略。
    sourceKey      使用语言文件时的来源文本 key，可省略。

限制：

    stepItems 外层数量必须等于 stars。
    每个阶段至少 1 个可提交物品。
    每个阶段最多 9 个可提交物品。
    prerequisites 可以写空数组 []。

添加中文研究示例：

    // 添加或替换一条研究。
    Research.addResearch(
        // 第 1 个参数：研究 id。
        // 研究笔记、完成研究、前置判断都会使用这个 id。
        "pack:test_research",

        // 第 2 个参数：研究显示名称。
        "测试研究",

        // 第 3 个参数：研究星级。
        // 必须等于 stepItems 外层数组数量。
        2,

        // 第 4 个参数：前置研究列表。
        // 空数组表示没有前置研究。
        [],

        // 第 5 个参数：每一级可提交物品。
        [
            // 第 1 级研究任务组。
            [
                // 可选提交项 1：提交 2 张纸。
                <minecraft:paper> * 2,
                // 可选提交项 2：提交书。
                <minecraft:book>
            ],

            // 第 2 级研究任务组。
            [
                // 可选提交项 1：提交灵魂碎片。
                <eidolon:soul_shard>
            ]
        ],

        // 第 6 个参数：来源文字。
        "用笔记工具右键钻石块。",

        // 第 7 个参数：描述文字。
        "这是由 CraftTweaker 2 添加的测试研究。"
    );

添加使用 lang key 的研究示例：

    // 添加一条使用 lang key 的研究。
    // 显示名会从语言文件 key 中读取，适合整合包自行维护 lang 文件时使用。
    Research.addResearchLang(
        // 研究 id。
        "pack:lang_research",
        // 研究星级。
        1,
        // 前置研究列表。
        [],
        // 每一级可提交物品。
        [
            [
                <minecraft:book>
            ]
        ],
        // 来源文本 lang key。
        "research_source.pack.lang_research"
    );

添加研究触发示例：

    // 方块触发。
    // 玩家用笔记工具右键黑曜石时，生成 pack:test_research 研究笔记。
    Research.addBlockTrigger("minecraft:obsidian", "pack:test_research");

    // 实体触发。
    // 玩家用笔记工具右键猪时，生成 pack:test_research 研究笔记。
    Research.addEntityTrigger("minecraft:pig", "pack:test_research");

    // 维度触发。
    // 玩家在主世界右键空气时，生成 pack:test_research 研究笔记。
    Research.addDimensionTrigger(0, "pack:test_research");

    // 流体触发。
    // 玩家用笔记工具右键静态水时，生成 pack:test_research 研究笔记。
    Research.addFluidTrigger("minecraft:water", "pack:test_research");

    // 流体触发。
    // 玩家用笔记工具右键流动水时，生成 pack:test_research 研究笔记。
    Research.addFluidTrigger("minecraft:flowing_water", "pack:test_research");

触发规则说明：

    同一个来源可以绑定多个研究。
    如果同一个来源绑定多个研究，玩家每次触发会轮流生成不同研究笔记。
    轮换进度按玩家保存。
    方块、实体、维度、流体的轮换互相独立。
    当前实现不检查玩家是否已经掌握该研究；触发成功就会生成笔记。

删除研究和触发示例：

    // 按研究 id 删除。
    Research.removeById("eidolon:gluttony");

    // 删除某个方块的全部研究触发。
    Research.removeBlockTriggers("minecraft:obsidian");

    // 删除某个实体的全部研究触发。
    Research.removeEntityTriggers("minecraft:pig");

    // 删除某个维度的全部研究触发。
    Research.removeDimensionTriggers(0);

    // 删除某个流体方块的全部研究触发。
    Research.removeFluidTriggers("minecraft:water");

    // 删除全部研究。谨慎使用。
    // Research.removeAll();

    // 删除全部研究触发。谨慎使用。
    // Research.removeAllTriggers();


八、Incubator：培养器配方
=======================

接口类：

    mods.eidolon.Incubator

导入：

    // 导入 EidolonLegacy 的 Incubator CT 接口。
    import mods.eidolon.Incubator;

用途：

    添加、删除培养器配方。

方法：

    Incubator.addRecipe(String id, IIngredient input, IIngredient catalyst, IItemStack output, @Optional int ticks);
    Incubator.removeById(String id);
    Incubator.removeByOutput(IIngredient output);
    Incubator.removeAll();

参数：

    id        培养器配方 id。
    input     原料，放在培养器上方输入槽。
    catalyst  催化物，放在培养器下方输入槽。
    output    输出物品。
    ticks     处理时间，单位为游戏刻。20 刻约等于 1 秒。
              省略或小于等于 0 时使用默认 1200 刻。

添加配方示例：

    // 删除同 id 的旧配方。
    Incubator.removeById("pack:test_incubator");

    // 添加一条培养器配方。
    Incubator.addRecipe(
        // 第 1 个参数：配方 id。
        "pack:test_incubator",

        // 第 2 个参数：原料。
        <minecraft:rotten_flesh>,

        // 第 3 个参数：催化物。
        <eidolon:soul_shard>,

        // 第 4 个参数：输出。
        <eidolon:zombie_heart>,

        // 第 5 个参数：处理时间。
        // 200 刻约等于 10 秒。
        200
    );

省略时间示例：

    // 不写 ticks 时使用默认处理时间。
    Incubator.addRecipe(
        // 配方 id。
        "pack:test_incubator_default_time",
        // 原料。
        <minecraft:bone>,
        // 催化物。
        <eidolon:death_essence>,
        // 输出。
        <minecraft:gold_nugget>
    );

删除配方示例：

    // 按配方 id 删除。
    Incubator.removeById("pack:test_incubator");

    // 按输出物品删除。
    Incubator.removeByOutput(<eidolon:zombie_heart>);

    // 删除全部培养器配方。谨慎使用。
    // Incubator.removeAll();


九、完整测试建议
================

1. 把测试脚本放到 scripts 目录中，例如：

       run/client/scripts/eidolon_ct2_smoke_test.zs

2. 启动客户端。

3. 查看：

       run/client/crafttweaker.log

4. 确认脚本加载日志中出现：

       Loading Eidolon CraftTweaker2 smoke test script
       Loaded Eidolon CraftTweaker2 smoke test script

5. 进游戏用 HEI 或 Codex 检查新增内容。

6. 如果某类内容不显示：

       先看 crafttweaker.log 是否有该行脚本报错。
       再看 latest.log 是否有注册或运行时报错。
       坩埚配方优先检查 stirs 数量是否等于 steps 数量。
       研究优先检查 stars 是否等于 stepItems 外层数量。
       Worktable 优先检查 grid 是否正好 9 个、reagents 是否正好 4 个。


十、正式整合包建议
==================

1. 不要把临时测试脚本长期放在正式包里。

2. 每个系统单独写一个 .zs 文件，便于定位错误，例如：

       scripts/eidolon/worktable.zs
       scripts/eidolon/crucible.zs
       scripts/eidolon/altar.zs
       scripts/eidolon/research.zs
       scripts/eidolon/incubator.zs

3. 替换内置内容时用 removeById 精确删除。

4. 只有在完全重写某个系统时才使用 removeAll。

5. 给自定义研究和仪式补语言 key，避免游戏内出现未汉化文本。

6. 修改后优先用一条小配方做验证，再批量加入完整内容。这样定位脚本错误会轻松很多。
